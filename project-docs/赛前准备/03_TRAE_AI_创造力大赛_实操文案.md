# TRAE AI 创造力大赛｜实操文案

## 你现在具体要干什么
### 第一步：先定题
题目先不要发散，主推就做一个**分享会筹备助手**。目标不是做大平台，而是做一个能演示完整闭环的小产品。

### 第二步：先定 demo
初赛最小 demo 只做 3 件事：
1. 输入一个分享准备目标
2. 自动拆成筹备任务
3. 输出阶段报告 / 结果摘要

### 第三步：先做最小版本
先别加复杂功能。先做出一个能跑、能演示、能截图、能录屏的版本。

## 推荐演示流程
### 场景示例
用户输入：
“我要准备一场小型分享会，但我不知道该怎么安排。”

工具输出：
- 自动拆成准备任务
- 给出执行顺序
- 生成阶段进度汇总
- 最后输出一份可交付的行动清单

## 记住
- 初赛目标不是做最复杂
- 初赛目标是做“最容易被看懂、最容易被演示、最容易被认可”的作品
- 你最终提交的不只是一个 demo，本质上还是两类社区内容：报名帖 + Demo 作品帖
- 官方明确强调：有过程、有体验、讲得清楚，就有机会晋级；不拼复杂度
- 所以从一开始就要留素材：截图、对话记录、版本过程、演示图、结果图
- 最稳的做法是：报名审核期间就提前打磨 Demo，但不要忘记“报名通过”才是 Demo 生效前提
- 已确认的大坑有五个：
  1. 只顾做作品，不按官方报名帖 / 初赛帖流程走
  2. 只交结果，不留创作真实性材料
  3. 发帖分区、标签、内容结构没对齐官方指南
  4. 没有体验链接或可体验入口，导致评委只能看描述不能实际体验
  5. 没有补齐关键步骤截图和 Session ID，真实性材料不完整
