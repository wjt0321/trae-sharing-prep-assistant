# TRAE AI 创造力大赛｜参赛方案草案

> 用途：这份只看“选题方向与优先级”，不是最终执行稿；真正开干时请优先看 `competition_notes/00_TRAE_AI_创造力大赛_总入口.md`。

## 方案 A：AI 任务管家
### 适合原因
- 最容易做出完整闭环
- 展示效果直观，评审容易理解
- 适合个人快速完成初赛

### 核心功能
- 输入目标，自动拆解任务
- 生成待办、日程、提醒
- 汇总进度，输出阶段报告

### 亮点
- 强产品感
- 可扩展到复赛的完整工作流
- 演示时很容易讲清楚价值

### 风险
- 容易做成普通聊天机器人
- 需要强调“执行流”和“结果产出”

## 方案 B：创意提案 / 文案生成器
### 适合原因
- 和赛事报名场景天然贴合
- 很适合做 Demo 和短视频演示
- 可快速加上模板、润色、导出

### 核心功能
- 输入一个想法，自动生成提案
- 一键输出 HTML / 文案 / 结构化大纲
- 支持多轮迭代和风格切换

### 亮点
- 贴近赛事
- 成本低、交付快
- 容易出效果图

### 风险
- 容易同质化
- 需要做出“比普通 AI 生成更好”的差异

## 方案 C：单场景工作流工具
### 适合原因
- 最像“正式产品”
- 适合复赛扩展
- 容易体现专业度

### 核心功能
- 盯住一个具体场景：报名、整理、汇总、复盘、发布
- 自动采集信息并生成可交付结果
- 支持历史记录与版本管理

### 亮点
- 产品边界清晰
- 复赛可持续迭代
- 更容易打出专业评审分

### 风险
- 题目选得不好会显得窄
- 前期需求定义要做准

## 我的建议
- 主方案选 **A：AI 任务管家**
- 备选方案选 **B：创意提案 / 文案生成器**

## 推荐切入点
- 做一个“能把一个想法变成可执行计划 + 可交付结果”的产品
- 初赛先做轻量版：输入目标 → 生成任务 → 输出结果
- 复赛再加：历史记录、协作、导出、版本迭代
