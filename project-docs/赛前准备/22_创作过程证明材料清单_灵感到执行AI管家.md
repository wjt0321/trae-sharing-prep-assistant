# 创作过程证明材料清单｜灵感到执行 AI 管家

## 目标
满足赛事对创作真实性的要求，确保后续如被抽查、复核或评审追问时，能够完整证明作品确实基于 TRAE Work / TRAE IDE 创作完成。

## 一、必须保留的材料
### 1. AI 对话截图
- 关键创意阶段
- 关键方案讨论阶段
- 页面与功能设计阶段
- 结果优化阶段

### 2. 任务历史 / 操作记录
- TRAE Work / TRAE IDE 中的重要任务历史
- 关键生成步骤和修改步骤
- 能体现项目逐步成型的操作轨迹

### 3. 关键 Session ID
- 与项目方向确认有关的 Session ID
- 与项目方案生成有关的 Session ID
- 与页面 / Demo 制作有关的 Session ID
- 与优化迭代有关的 Session ID

### 4. 阶段成果截图
- 项目方案文档
- 提案文稿
- 页面草图
- Demo 初版
- Demo 优化版
- 最终可提交页面

### 5. 创作过程说明
- 作品是如何从想法演化到方案的
- AI 在哪些环节参与最深
- 哪些地方进行了多轮修改与优化

## 二、建议补充的材料
### 6. 页面版本记录
- 初版页面截图
- 二次迭代页面截图
- 最终版页面截图

### 7. 输出结果对比
- 初次生成方案截图
- 追加条件后的优化版截图
- 用来体现 AI 的动态调整能力

### 8. 演示视频 / 录屏素材
- 录制主流程演示
- 录制二次优化过程
- 可用于发帖或后续复核

## 三、推荐整理方式
### 文件夹建议
- `proof/01_对话截图`
- `proof/02_任务历史`
- `proof/03_session_id记录`
- `proof/04_阶段成果截图`
- `proof/05_版本对比`
- `proof/06_演示录屏`

### 记录原则
- 每完成一个阶段就保存一次
- 不要等到最后统一补
- 优先保留“关键转折点”和“明显版本变化”

## 四、最容易遗漏的点
- 只留了最终成品，没有留中间过程
- 截图很多，但没有时间线和说明
- 没有专门记录关键 Session ID
- 没有保留 AI 参与页面与产品结构设计的证据

## 五、当前就该做的动作
- 从现在开始保存关键截图
- 把每次重要讨论记下时间和主题
- 专门整理一个 Session ID 清单
- 每次页面有明显变化就截图留档
