# 最终检查清单

## 目标

这份清单用于比赛项目提交前的最后核对。

原则：

- 不再讨论方向
- 不再发散需求
- 只检查是否已经具备交卷条件

---

## 一、产品完成度检查

- [ ] 首页可正常打开
- [ ] 输入区可正常填写
- [ ] 示例按钮可正常使用
- [ ] 结果生成流程完整
- [ ] 四阶段卡片正常显示
- [ ] 分享结构建议正常显示
- [ ] 行动清单正常显示
- [ ] 准备重点 / 风险提醒正常显示
- [ ] 页面整体没有明显错位或断裂
- [ ] 页面整体风格保持统一

---

## 二、逻辑与内容检查

- [ ] 不同输入能产生不同结果重点
- [ ] 输出内容没有明显空字段或异常文案
- [ ] 结果仍然像“可执行方案”而不是泛泛建议
- [ ] 行动清单有优先级感
- [ ] 阶段之间有顺序感
- [ ] 提醒区内容像经验判断，而不是凑数文案

---

## 三、演示材料检查

- [ ] 演示脚本已准备
- [ ] 1 分钟版本可直接使用
- [ ] 3 分钟版本可直接使用
- [ ] 推荐示例输入已确定
- [ ] 演示顺序已固定
- [ ] 演示时的收口句已确定

---

## 四、答辩材料检查

- [ ] 一句话项目定义已确定
- [ ] 标准作品简介已确定
- [ ] 亮点包装版本已确定
- [ ] 答辩口播稿已准备
- [ ] 评审 Q&A 已准备
- [ ] 常见问题回答口径统一

---

## 五、素材检查

- [ ] 首屏截图已准备
- [ ] 结果总览截图已准备
- [ ] 四阶段区截图已准备
- [ ] 行动清单区截图已准备
- [ ] 提醒区截图已准备
- [ ] 录屏已完成或录屏方案已确认
- [ ] 素材命名清楚，不混乱

---

## 六、文档与归档检查

- [ ] 项目正式文档已归档
- [ ] 未来方向文档已补齐
- [ ] 比赛相关输出都在 `D:\competition_notes` 下
- [ ] 文档命名清楚，便于快速打开
- [ ] 提交时要用的文件已经明确

---

## 七、风险与合规检查

- [ ] 没有内部身份称谓
- [ ] 没有不适合公开的信息
- [ ] 没有明显会引起误解的措辞
- [ ] 页面里没有临时占位内容
- [ ] 没有未清理的测试/演示残留信息

---

## 八、最终确认

当以下三条都成立时，才可以正式判断这个比赛项目进入可交付状态：

- [ ] 产品可演示
- [ ] 材料可说明
- [ ] 答辩可应对

如果这三条都成立，就说明项目已经不只是“做完了”，而是“可以交卷了”。
