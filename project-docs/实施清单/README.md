# 实施清单目录

## 说明

本目录基于 `项目文档/08_未来成品蓝图_从分享会筹备助手到AI任务管家.md` 拆分而来。

拆分原则：

- 一个实施目标一份清单
- 每份清单都可独立查阅与单独推进
- 每份清单都包含目标、范围、前后端设计重点、数据与 AI 影响、验收标准和依赖关系
- 总体顺序遵循“先搭产品骨架，再补核心能力，最后补协作、自动化和治理”的路线

适用方式：

- 如果要整体推进，按编号顺序阅读
- 如果要做某个模块，直接打开对应目标清单
- 如果后续进入开发，可把单份清单继续细化为任务单、里程碑和版本计划

## 使用建议

如果后续要正式进入开发，建议再从本目录继续往下拆两层：

- 第一层：按目标建立版本范围
- 第二层：按版本建立开发任务单

例如：

- `04_AI规划与重规划引擎实施清单.md`
  - 可继续拆成 `V1 首版规划`、`V1.1 重规划`、`V1.2 方案比较`

这样后面无论是自己迭代，还是交给 AI / 协作者推进，都更容易调取与执行。
