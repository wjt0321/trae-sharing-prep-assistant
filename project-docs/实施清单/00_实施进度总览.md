### 关键技术约定（踩坑记录）

1. **后端启动**：`pnpm run start:dev`（开发模式，watch）或 `node dist/main.js`（生产模式，需先 build）。tsconfig.build.json 已设置 `incremental: false` 解决 nest build 不生成 dist 的问题
2. **PowerShell 限制**：不支持 heredoc（`<<'EOF'`），commit message 用多个 `-m` 参数
3. **git 代理**：push 前设置 socks5h://127.0.0.1:10808，push 后必须 unset（否则影响其他网络操作）
4. **共享包构建**：后端依赖 `packages/shared/dist/`，shared 改动后必须 `pnpm run build`
5. **Prisma 迁移**：新增模型/字段后 `pnpm run db:migrate`，迁移名格式 `YYYYMMDDHHMMSS_描述`
6. **跨模块依赖**：CollaborationModule 被 PlanningModule 和 ExecutionModule import（记录活动流），新增通知模块也应 import CollaborationModule
7. **工作区成员 API**：返回扁平结构 `{ userId, displayName, role, ... }`（不是嵌套 `{ user: {...}, role }`）
8. **Plan.changeReason**：替代从 aiPromptLog 解析 replanReason，replanReason 作为兼容字段保留
9. **@Public 装饰器**：用于公开共享页（无需 JWT），在 `apps/server/src/presentation/` 中定义
10. **前端 API 代理**：`/api/server/*` 通过 rewrites 代理到后端 `http://localhost:4000`
11. **设计系统 token**：在 `apps/web/src/app/globals.css` 的 `@theme inline` 中定义（无 tailwind.config.js）
12. **任务状态机**：TASK_STATUS_TRANSITIONS 定义合法转换，blocked 必填 blockerNote
13. **ResponseInterceptor 字段冲突**：拦截器检查 `'success' in data`，若返回值含 `success` 字段会跳过包装直接返回。DTO 中表示"是否成功"应使用 `connected` / `cleared` / `ok` 等替代字段名
14. **AI 配置加密**：API Key 用 AES-256-GCM 加密存储（`iv:authTag:ciphertext` 格式），加密 key 来自 `AI_CONFIG_ENCRYPTION_KEY` 环境变量（缺省回退到 `JWT_SECRET`）。读取时脱敏（`prefix****suffix`），仅 AiGateway 实际调用时解密。配置缓存 30s TTL，更新后立即失效
15. **AiGateway mock 回退**：无配置或真实调用失败时自动回退到 mock，主流程不中断（错误记入 AiCallLog）
16. **SSE 流式响应**：用 `@Res()` 装饰器注入 Express Response 直接写入，绕过 ResponseInterceptor（拦截器会包装 `{success, data, ...}` 不适合 SSE）。设置 `Content-Type: text/event-stream` + `Cache-Control: no-cache` + `Connection: keep-alive` + `X-Accel-Buffering: no`，每条消息格式 `data: {json}\n\n`，结束时 `data: {"done": true}\n\n`。前端用 `fetch` + `ReadableStream` reader 解析（不用 EventSource，因为要带 Authorization header）
17. **Prompt Registry 版本管理**：PromptTemplate 用 `@@unique([name, version])`，同名只能有一个 `isActive=true`。`create` 若 name 已存在则创建新版本（version+1）并自动激活旧版本设为非活跃；`update` 总是创建新版本；`activate` 切换活跃版本；`delete` 后自动激活剩余最新版本。调用方用 `renderForCall(name, variables)` 解析模板，返回 `{ messages, version }` 或 `null`（未找到时调用方用内联 fallback）
18. **Prompt 变量渲染**：模板用 `{{var}}` 占位符，正则 `/\{\{(\w+)\}\}/g` 替换。DTO 中 `variables: Record<string, string>` 用 `@IsObject()` 校验（不是 `@IsArray()`）。嵌套 DTO 数组用 `@ValidateNested({ each: true })` + `@Type(() => XxxDto)`
19. **DeepSeek 默认示例**：AI 配置页 `PROVIDER_PRESETS` 默认示例使用 DeepSeek（`https://api.deepseek.com` + `deepseek-v4-flash`），Anthropic 兼容代理用 `https://api.deepseek.com/anthropic`。用户可自行替换为其他服务商，URL/Key/模型名均不硬编码在代码逻辑中（仅在 UI preset 中作为示例默认值）
20. **成本统计定价表**：`MODEL_PRICING` 在 `ai-config.service.ts` 末尾，每 1M token 价格（input/output）。未列出的模型（含 `mock-model` 和未配置的 DeepSeek v4 模型）不计成本（返回 0）。新增模型需手工添加到定价表
