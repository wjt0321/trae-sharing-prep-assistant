# 10_设计系统与核心前端体验实施清单

## 目标

把 PRD 中定义的 UI/UX 规范落成真正可复用的前端体验体系，确保未来产品扩展后依然保持清晰、克制、有产品感。

---

## 交付范围

本目标覆盖：

- 设计系统 token
- 基础组件规范
- 页面框架规范
- 核心页面原型
- 状态覆盖规范
- 设计验收规则

本目标不直接覆盖：

- 具体业务 API
- AI 提示词逻辑

---

## 主要实施项

### 1. 建立设计系统 token

- 色彩 token
- 字体 token
- 间距 token
- 圆角 token
- 阴影 token
- 状态色 token

### 2. 建立基础组件库

至少覆盖：

- 按钮
- 输入框
- 选择器
- 标签
- 卡片
- 空状态
- 错误态
- Skeleton
- Toast
- Modal

### 3. 建立页面骨架规范

- 首页框架
- 应用页框架
- 工作台框架
- 详情页框架
- 设置页框架

### 4. 建立关键页面原型

按 PRD 要求，至少输出：

- 首页
- 目标创建器
- 规划面板
- 执行工作台
- 模板库

### 5. 落地 `huashu-design` 与 `open-design`

必须纳入：

- 高保真 HTML 原型优先
- 设计方向变体探索
- 反 AI slop 规则
- 排版规则
- 五态覆盖规则
- 自动化点击验证

### 6. 建立设计验收清单

- 是否像产品而不是模板
- 是否覆盖所有必要状态
- 是否避免聊天页套壳
- 是否具有统一的工作台逻辑

---

## 设计规范重点

### 来自 `huashu-design`

- 先原型再定稿
- 模糊阶段至少给 3 个方向
- 关键流程做交互验证
- 重要页面可输出演示用版本

### 来自 `open-design`

- 避免默认 AI 视觉套路
- 严格控制排版层级和字距
- 任何交互面都必须有 `Loading / Empty / Error / Populated / Edge`
- 设计要能被稳定工程化落地

---

## 前端实现重点

- 使用 `Next.js + Tailwind CSS + shadcn/ui`
- 建立统一 `layout`、`section`、`card`、`panel` 约定
- 保证组件状态完整
- 保证多页面扩展时视觉不失控

## 完成标准

- 已有一套可复用设计 token
- 已有基础组件规范
- 已有关键页面原型
- 已有明确状态覆盖规则
- 后续业务页面可以基于这套系统快速扩展

## 建议拆分顺序

1. 先做 token
2. 再做组件库
3. 再做页面骨架
4. 再做关键页面原型
5. 最后做设计验收与交互验证
