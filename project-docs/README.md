# project-docs 目录说明

## 目录结构

- `赛前准备`
  - 归档本项目现有的赛前思考、方案草稿、比赛资料、执行提示词与历史文档。
- `初赛文档`
  - 存放把项目从“展示型 Demo”推进到“实战 MVP”的规划、实施步骤与代码级任务单。
- `项目文档`
  - 存放与当前项目成品直接对应的正式材料，例如作品简介、截图点位、演示口播稿、首页文案定稿、MVP 改造总结等。
- `未来方向`
  - 存放 MVP 完成之后的比赛交付收口文档，例如完赛标准、演示脚本、答辩话术、评审 Q&A、素材清单、最终检查表及与初赛文档的对应关系。

## 当前约定

- 原有项目文档已从 `docs/` 迁移到 `project-docs/`
- `docs/` 目录专用于 GitHub Pages 静态发布产物
- 与 **MVP 落地实现** 直接相关的规划和改造文档，统一放在 `project-docs/初赛文档`
- 与 **当前项目成品** 直接对应、可用于对外展示或对内归档的正式材料，统一放在 `project-docs/项目文档`
- 与 **比赛交付收口** 直接相关的文档，统一放在 `project-docs/未来方向`

## 推荐阅读顺序

1. 先看 `初赛文档/`，理解项目如何从 Demo 升级为实战 MVP
2. 再看 `项目文档/`，了解当前成品材料与对外表达
3. 最后看 `未来方向/`，处理比赛交付收口、答辩和最终检查
