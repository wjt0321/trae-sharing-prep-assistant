# project-docs 目录说明

## 目录结构

- `赛前准备`
  - 归档本项目现有的赛前思考、方案草稿、比赛资料、执行提示词与历史文档。
- `初赛文档`
  - 存放把项目从“展示型 Demo”推进到“实战 MVP”的规划、实施步骤与代码级任务单。
- `项目文档`
  - 存放与当前项目成品直接对应的正式材料，例如作品简介、截图点位、演示口播稿、首页文案定稿、MVP 改造总结，以及未来成品蓝图 / 长期产品规划等。
- `实施清单`
  - 存放基于长期 PRD 拆分出的多份目标级实施清单，一份目标一份文件，便于后续按模块调取和推进。
- `未来方向`
  - 存放 MVP 完成之后的比赛交付收口文档，例如完赛标准、演示脚本、答辩话术、评审 Q&A、素材清单、最终检查表及与初赛文档的对应关系。

## 当前约定

- 原有项目文档已从 `docs/` 迁移到 `project-docs/`
- `docs/` 目录专用于 GitHub Pages 静态发布产物
- 与 **MVP 落地实现** 直接相关的规划和改造文档，统一放在 `project-docs/初赛文档`
- 与 **当前项目成品** 直接对应、可用于对外展示或对内归档的正式材料，以及未来成品蓝图 / 长期产品规划，统一放在 `project-docs/项目文档`
- 与 **长期产品落地实施** 直接相关、按目标拆分的任务型文档，统一放在 `project-docs/实施清单`
- 与 **比赛交付收口** 直接相关的文档，统一放在 `project-docs/未来方向`

## 推荐阅读顺序

1. 先看 `初赛文档/`，理解项目如何从 Demo 升级为实战 MVP
2. 再看 `项目文档/`，了解当前成品材料、对外表达与长期产品蓝图
3. 再看 `实施清单/`，按目标逐份调取长期产品落地任务
4. 最后看 `未来方向/`，处理比赛交付收口、答辩和最终检查

如果你想直接看项目长期产品终局，请优先阅读：

- `项目文档/08_未来成品蓝图_从分享会筹备助手到AI任务管家.md`

如果你想直接进入后续执行，请优先阅读：

- `实施清单/README.md`
