# 部署指南：Vercel + Railway

将 AI 任务管家从本地 SQLite 单机部署迁移到云端全栈部署。

## 架构概览

```
┌─────────────────────────┐     ┌─────────────────────────┐
│   Vercel（前端）         │     │   Railway（后端）        │
│   Next.js App Router    │────▶│   NestJS + Prisma       │
│   https://xxx.vercel.app│     │   https://xxx.up.railway│
└─────────────────────────┘     └──────────┬──────────────┘
                                           │
                                ┌──────────▼──────────────┐
                                │   Railway PostgreSQL     │
                                │   持久化数据库            │
                                └─────────────────────────┘
```

| 组件 | 平台 | 说明 |
|---|---|---|
| 前端 | Vercel | Next.js 原生支持，免费 tier 够用 |
| 后端 | Railway | NestJS 容器化部署，免费 trial $5 额度 |
| 数据库 | Railway PostgreSQL | 与后端同平台，内网通信零延迟 |

## 前置条件

1. **GitHub 仓库**：代码已推送到 `https://github.com/wjt0321/trae-sharing-prep-assistant`
2. **Vercel 账号**：https://vercel.com （用 GitHub 登录，免费）
3. **Railway 账号**：https://railway.app （用 GitHub 登录，免费 trial）
4. **本地环境**：pnpm 10 + Node 20+，用于修改 Prisma 配置

## 第一步：数据库迁移（SQLite → PostgreSQL）

Railway 不支持 SQLite 文件持久化，需切换为 PostgreSQL。

### 1.1 修改 Prisma schema

编辑 `apps/server/prisma/schema.prisma`：

```prisma
datasource db {
  provider = "postgresql"   // 从 "sqlite" 改为 "postgresql"
  url      = env("DATABASE_URL")
}
```

### 1.2 创建 PostgreSQL 迁移

本地安装 PostgreSQL 或使用 Docker 临时启动：

```bash
# 临时启动 PostgreSQL（可选，仅用于生成迁移文件）
docker run --name pg-temp -e POSTGRES_PASSWORD=temp -p 5432:5432 -d postgres:16

# 设置临时数据库连接
set DATABASE_URL="postgresql://postgres:temp@localhost:5432/temp?schema=public"

# 生成新迁移
cd apps/server
pnpm run db:generate
pnpm run prisma:migrate -- --name init_postgresql --create-only

# 验证迁移文件生成
dir prisma\migrations
```

### 1.3 提交迁移变更

```bash
git add apps/server/prisma/schema.prisma apps/server/prisma/migrations/
git commit -m "feat(db): 切换 Prisma provider 为 PostgreSQL 以支持云端部署"
git push origin master
```

> **注意**：切换 provider 后，本地开发也需用 PostgreSQL。可用 Docker 启动本地 PG，
> 或保留两个 schema 文件分支（不推荐）。

## 第二步：Railway 部署（后端 + 数据库）

### 2.1 创建 Railway 项目

1. 登录 https://railway.app → **New Project**
2. 选择 **Provision PostgreSQL** → 自动创建数据库服务
3. 在数据库服务的 **Connect** 标签页，复制 **PostgreSQL Connection URL**
   （格式：`postgresql://postgres:xxx@xxx.railway.app:5432/railway`）

### 2.2 添加后端服务

1. 同一项目内 → **New Service** → **GitHub Repo**
2. 选择 `wjt0321/trae-sharing-prep-assistant` 仓库
3. 配置服务：

| 配置项 | 值 |
|---|---|
| **Root Directory** | `apps/server` |
| **Build Command** | `cd ../.. && pnpm install --frozen-lockfile && pnpm --filter @ai-task-manager/shared build && pnpm --filter @ai-task-manager/server build` |
| **Start Command** | `pnpm run start:prod` |
| **Watch Path** | `apps/server/**,packages/shared/**` |

### 2.3 配置后端环境变量

在 Railway 后端服务的 **Variables** 标签页添加：

| 变量名 | 值 | 说明 |
|---|---|---|
| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` | 引用同项目 PostgreSQL 服务（Railway 自动注入） |
| `JWT_SECRET` | `<生成 32+ 字节随机字符串>` | 生产密钥，务必强随机 |
| `JWT_EXPIRES_IN` | `1d` | |
| `JWT_REFRESH_EXPIRES_IN` | `7d` | |
| `CORS_ORIGIN` | `https://xxx.vercel.app` | Vercel 部署后填入实际域名 |
| `AI_GATEWAY_PROVIDER` | `mock` | 初始用 mock，后续可在网页配置真实 AI |
| `UPLOAD_DIR` | `./uploads` | 临时文件存储（见已知限制） |
| `TASK_WORKER_INTERVAL_MS` | `5000` | |
| `RATE_LIMIT_PER_MINUTE` | `120` | |
| `RATE_LIMIT_AUTH_PER_MINUTE` | `10` | |
| `AUDIT_LOG_RETENTION_DAYS` | `90` | |

> **JWT_SECRET 生成方式**：`node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

### 2.4 配置数据库迁移（Release 阶段）

在 Railway 后端服务 → **Settings** → **Deploy**：

将 **Start Command** 改为：

```bash
cd ../.. && pnpm --filter @ai-task-manager/server exec prisma migrate deploy && pnpm --filter @ai-task-manager/server exec prisma db seed && pnpm --filter @ai-task-manager/server start:prod
```

这会在每次部署时：
1. `prisma migrate deploy` — 应用数据库迁移
2. `prisma db seed` — 写入种子数据（首次部署需要，后续可移除）
3. `start:prod` — 启动 NestJS 服务

### 2.5 配置健康检查

Railway → 后端服务 → **Settings** → **Healthcheck**：

| 配置项 | 值 |
|---|---|
| **Path** | `/health` |
| **Port** | `4000`（或 Railway 自动分配的 `PORT`） |

> Railway 会自动注入 `PORT` 环境变量。后端 `main.ts` 已读取 `PORT`，
> 但需确认 Railway 的端口映射。如果健康检查失败，检查 `PORT` 是否被正确读取。

### 2.6 获取后端公网地址

部署成功后，Railway 会分配公网域名：
- **Settings** → **Networking** → **Generate Domain**
- 格式：`https://xxx-production.up.railway.app`
- 记下这个地址，Vercel 配置需要用

## 第三步：Vercel 部署（前端）

### 3.1 导入项目

1. 登录 https://vercel.com → **Add New** → **Project**
2. 选择 `wjt0321/trae-sharing-prep-assistant` 仓库
3. Vercel 会自动检测 Next.js，但需手动调整配置

### 3.2 配置构建设置

| 配置项 | 值 |
|---|---|
| **Framework Preset** | Next.js |
| **Root Directory** | `apps/web` |
| **Build Command** | `cd ../.. && pnpm install --frozen-lockfile && pnpm --filter @ai-task-manager/shared build && pnpm --filter @ai-task-manager/web build` |
| **Output Directory** | `.next` （自动检测） |
| **Install Command** | `pnpm install` （自动检测） |

### 3.3 配置环境变量

在 Vercel 项目的 **Settings** → **Environment Variables**：

| 变量名 | 值 | 说明 |
|---|---|---|
| `SERVER_BASE_URL` | `https://xxx-production.up.railway.app` | Railway 后端公网地址（第二步获取） |

> **原理**：`next.config.ts` 的 `rewrites()` 读取 `SERVER_BASE_URL`，
> 将 `/api/server/*` 代理到后端。本地默认 `http://localhost:4000`，
> Vercel 上需指向 Railway 后端。

### 3.4 部署

点击 **Deploy**，等待构建完成（约 2-3 分钟）。

部署成功后获得前端地址：`https://xxx.vercel.app`

### 3.5 回填 CORS_ORIGIN

**关键步骤**：前端部署后，回到 Railway 后端服务：

1. **Variables** → 将 `CORS_ORIGIN` 改为 Vercel 实际域名
2. 例如：`https://ai-task-manager.vercel.app`
3. Railway 会自动重新部署后端

## 已知限制与后续改进

### 1. 文件存储（uploads/）不持久

Railway 容器文件系统是临时的，每次重新部署 `uploads/` 目录会清空。

**当前影响**：导出文件、附件上传等功能生成的文件会丢失。

**后续改进方案**：
- 接入 S3 / Cloudflare R2 / 阿里云 OSS
- 修改 `StorageService` 从本地读写改为对象存储 SDK
- 优先级：中（比赛演示不涉及大量文件上传）

### 2. AI 网关配置不持久

AI 网关的 API Key 存在数据库 `AiProviderConfig` 表（AES-256-GCM 加密），
数据库是持久的，所以 **AI 配置会保留**。但加密密钥如果依赖环境变量，
需确保 `AI_GATEWAY_ENCRYPTION_KEY`（如有）在 Railway Variables 中配置。

### 3. 数据库种子数据

首次部署会执行 `prisma db seed` 写入演示数据。后续部署如果不想重复写入，
可将 Start Command 中的 `prisma db seed` 移除（seed 使用 upsert，重复执行幂等，
不移除也没问题）。

### 4. Railway 免费额度

Railway 免费 trial 提供 $5 信用额度，约可运行：
- 后端服务：约 30 天（512MB 内存）
- PostgreSQL：约 20 天（1GB 存储）

**超出后**：需绑定信用卡按量付费，或迁移到 Render 免费 tier（会休眠）。

### 5. Vercel 免费额度

Vercel Hobby（免费）tier：
- 100GB 带宽/月
- 100 小时构建/月
- 对个人项目足够

## 快速排障

| 问题 | 排查方向 |
|---|---|
| 前端 API 404 | 检查 `SERVER_BASE_URL` 是否正确指向 Railway 后端 |
| 前端 API CORS 错误 | 检查后端 `CORS_ORIGIN` 是否包含 Vercel 域名 |
| 后端启动失败 | 查看 Railway Deploy Logs，确认 `DATABASE_URL` 已注入 |
| 数据库迁移失败 | 确认 `prisma migrate deploy` 在 Start Command 中 |
| 健康检查失败 | 确认 `/health` 路径（无 `/api` 前缀）和端口配置 |
| 注册/登录 500 | 检查 `JWT_SECRET` 是否已设置 |
| AI 功能返回 mock | 正常现象，需在 `/app/settings/ai-config` 配置真实模型 |

## 部署后配置真实 AI

部署成功后，AI 默认是 mock 模式。要启用真实 AI：

1. 登录应用 → `/app/settings/ai-config`
2. 选择服务商（OpenAI 兼容 / Anthropic / 自定义）
3. 填入 Base URL + API Key + 模型名
4. 点"测试连接"验证
5. 点"流式试用"验证 SSE

配置会加密存入数据库，后续创建目标时的场景识别和规划生成都会调用真实大模型。
