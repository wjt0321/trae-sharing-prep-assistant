# 12 安全监控指标与发布治理 - 开发文档

> 本文档记录任务 12 的开发实现细节，包括审计日志、监控指标、速率限制、CI/CD、数据备份、安全扫描等模块的设计与实现。

## 1. 审计日志模块（audit）

### 1.1 数据模型

**AuditLog**（`apps/server/prisma/schema.prisma`）：

```prisma
model AuditLog {
  id            String   @id @default(uuid())
  actorId       String?  // 操作者 ID（未登录时为 null）
  actorEmail    String?  // 操作者邮箱（未登录时为 null，用于失败登录记录）
  action        String   // 动作类型（AuditActionEnum）
  resourceType  String?  // 资源类型（AuditResourceTypeEnum）
  resourceId    String?  // 资源 ID
  result        String   // 结果（success / failure）
  errorMessage  String?  // 失败时的错误信息
  ipAddress     String?  // 请求 IP
  userAgent     String?  // User-Agent
  method        String?  // HTTP 方法
  path          String?  // 请求路径
  detail        Json?    // 额外详情（JSON）
  createdAt     DateTime @default(now())

  @@index([actorId])
  @@index([action])
  @@index([result])
  @@index([createdAt])
}
```

### 1.2 审计动作枚举

```typescript
enum AuditActionEnum {
  LOGIN = 'login',
  LOGOUT = 'logout',
  REGISTER = 'register',
  CREATE = 'create',
  UPDATE = 'update',
  DELETE = 'delete',
  EXPORT = 'export',
  SHARE = 'share',
  CONFIG_CHANGE = 'config_change',
  PASSWORD_CHANGE = 'password_change',
  INVITE = 'invite',
  ROLE_CHANGE = 'role_change',
}
```

### 1.3 核心服务

**AuditService**（`apps/server/src/modules/audit/audit.service.ts`）：

- `record(params: CreateAuditLogParams)`：异步记录审计日志（fire-and-forget，不阻断主流程）
- `findAll(query)`：分页查询审计日志（支持 action/result/keyword 筛选）
- `findOne(id)`：查询单条审计日志
- `getStats(days)`：审计统计（按动作/资源类型/日期聚合 + 最近失败列表）
- `cleanup(retentionDays)`：清理过期审计日志（默认 90 天）

### 1.4 审计日志集成点

| 模块 | 集成点 | 动作 |
|------|--------|------|
| auth | register | REGISTER（成功/失败） |

### 1.5 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/audit` | 审计日志列表（分页 + 筛选） |

## 2. 监控指标模块（monitoring）

### 2.1 健康检查

**MonitoringService.checkHealth()**（`apps/server/src/modules/monitoring/monitoring.service.ts`）：

检查三个维度：
1. **database**：Prisma 连接测试
2. **disk**：数据库文件大小（>1GB degraded，>10GB unhealthy）
3. **task-backlog**：任务积压（>100 degraded，>500 unhealthy）

返回 `HealthResponseDto`：
```typescript
{
  status: 'ok' | 'degraded' | 'unhealthy',
  service: 'ai-task-manager-server',
  version: '0.1.0',
  env: 'development',
  timestamp: string,
  startedAt: string,
  uptimeSeconds: number,
  checks: HealthCheckResultDto[]
}
```

### 2.2 系统指标

**MonitoringService.getSystemMetrics(days)**：

- **requests**：请求数/错误率/慢请求/平均耗时（内存级统计，重启后丢失）
- **taskQueue**：任务队列状态（queued/running/succeeded/failed）
- **aiCalls**：AI 调用统计（从 AiCallLog 表聚合）
- **database**：数据库表统计（各表记录数）

### 2.3 业务指标

**MonitoringService.getBusinessMetrics(days)**：

17 个并行 Prisma 查询聚合：
- **goalCreation**：目标创建数/成功率/按场景类型/按日期
- **planAdoption**：规划采纳率（有规划的目标数/总目标数）
- **replanUsage**：重规划使用率（有重规划的目标数/总目标数）
- **goalProgress**：目标进度（按阶段/完成率/平均任务完成率）
- **exportShare**：导出分享统计（按类型/分享数/失败数）
- **templateReuse**：模板复用率（使用次数/平均使用/Top 5 模板）

### 2.4 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/monitoring/health` | 健康检查 |
| GET | `/api/monitoring/system?days=7` | 系统指标 |
| GET | `/api/monitoring/business?days=7` | 业务指标 |

所有接口均需 JwtAuthGuard。

## 3. 速率限制中间件

**RateLimitMiddleware**（`apps/server/src/presentation/rate-limit.middleware.ts`）：

- **默认限制**：120 次/分钟/IP
- **认证接口**：10 次/分钟/IP（`/api/auth/login`、`/api/auth/register`）
- **实现**：内存级 Map<IP, {count, resetAt}>
- **清理**：每 5 分钟清理过期记录
- **响应头**：`X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`
- **超限响应**：429 Too Many Requests

## 4. 请求日志中间件

**RequestLogMiddleware**（`apps/server/src/presentation/request-log.middleware.ts`）：

- 记录：method / path / statusCode / durationMs
- 写入 MonitoringService 内存统计
- 跳过 `/health` 路径
- 慢请求告警（>1s 输出 WARN 日志）

## 7. 安全扫描脚本

**scripts/security-scan.mjs**（`apps/server/scripts/security-scan.mjs`）：

检查项：
1. 硬编码 API Key / Secret（OpenAI / Anthropic / AWS / 通用模式）
2. .env 文件是否被 git 跟踪
3. JWT_SECRET 是否仍为默认值（please-change-me）
4. 明文密码赋值
5. 日志中打印敏感字段（apiKey / password / secret / token / authorization）

退出码：0 = 通过；1 = 发现 FAIL 级别问题

用法：`pnpm run security-scan`

## 10. 已知限制

1. **磁盘检查路径**：`/monitoring/health` 的 disk 检查在 SQLite 路径为相对路径时可能无法读取文件大小
2. **内存级速率限制**：单实例内存计数，多实例部署需替换为 Redis
3. **审计日志异步写入**：fire-and-forget，极端情况下可能丢失少量日志
4. **CI lint 非阻断**：CI 中 lint 设为 `continue-on-error: true`
5. **RBAC 未细化**：监控/审计接口仅要求登录，未区分管理员角色
