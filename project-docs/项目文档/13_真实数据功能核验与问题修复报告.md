## 4. 发现问题与修复

### 问题 1：文档产物路径不符合项目约束

现象：

- 早期规格文档曾落在 `docs/superpowers` 目录
- 该目录属于静态展示产物目录，不适合作为项目资料主存放位置

原因：

- 初始文档生成路径没有完全遵循仓库内的文档归档规则

处理结果：

- 项目资料统一迁移到 `project-docs/项目文档`
- 不再向 `docs/superpowers` 生成项目文档
- 当前正式文档已统一放在 `project-docs/项目文档`

### 问题 2：模板库页面加载失败

现象：

- 登录后访问 `/app/templates`
- 页面显示“加载模板失败”
- 页面请求返回 400

根因：

- 前端模板列表请求把 `workspaceId` 同时放在路径参数和查询参数中
- 后端模板查询 DTO 仅接受 `category / scenarioType / keyword / builtInOnly`
- 额外传入 `workspaceId` 会触发白名单校验，导致请求被拒绝

涉及文件：

- `apps/web/src/app/app/templates/page.tsx`
- `apps/server/src/modules/knowledge/dto/knowledge.dto.ts`

修复方式：

- 模板列表请求改为只拼接后端允许的筛选字段
- 当筛选为空时，不再附带多余查询串

修复结果：

- `/app/templates` 已恢复正常加载
- 页面可正常展示真实模板数据
- 浏览器走查中未再出现此前的 400 异常提示
