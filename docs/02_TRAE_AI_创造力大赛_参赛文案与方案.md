# TRAE AI 创造力大赛｜参赛文案与方案

## 当前主推方案
主推方案不再是泛泛的“AI 任务管家”，而是把它收窄成一个**单场景、强演示、强结果导向**的作品：

**分享会筹备助手**

一句话定义：
把“我要办一场分享会，但不知道怎么开始”这类模糊目标，自动拆成可执行的准备步骤，并输出一份可直接照着做的行动清单。

## 为什么改成这个包装
- 比“AI 任务管家”更具体，评委一眼就知道它在解决什么问题
- 更像真实产品，不像空泛概念
- 演示输入、演示过程、演示结果都更自然
- 更容易做出高质量截图和录屏
- 初赛先打穿一个场景，比做泛化工具更稳

## 这个方向为什么适合参赛
- 最容易做成完整闭环
- 展示效果直观，评审容易理解
- 适合个人快速完成初赛
- 有真实问题场景，不是空泛概念
- 既能先做最小 Demo，也保留复赛扩展空间

## 作品定义
这不是一个泛泛陪聊的聊天机器人，而是一个把“目标”转成“行动”的执行型工具。

当前建议的首发场景是：**准备一场小型分享会**。

用户输入一个分享会准备目标后，系统会：
- 理解这次分享会要做什么
- 自动拆解成可执行任务
- 给出准备顺序与重点提醒
- 输出阶段汇总或最终行动清单

如果后续要扩展，再把同样的能力迁移到学习计划、短期项目、活动筹备等场景。
## 要解决的问题
很多人不是没有目标，而是不知道第一步怎么开始，也不知道中间该怎么拆、怎么排、怎么收口。

这个作品要解决的是：
- 面对一个目标时，起步困难
- 任务拆解容易混乱
- 执行过程中缺少阶段汇总
- 最后难以形成清晰的结果输出

## 目标用户 / 使用场景
适合下面这类人：
- 想做一场分享会，但不知道怎么准备的人
- 需要快速整理思路的学生 / 职场人 / 个体创作者
- 想把一个临时目标快速整理成执行计划的人

适合演示的场景优先固定为：
- 准备一场小型分享会

后续扩展场景再考虑：
- 安排一次学习计划
- 整理一个临时项目的执行步骤
- 策划一次短途出行
## 初赛最小可交付版本
初赛最小 Demo 只做 3 件事：
1. 输入一个目标
2. 自动拆成任务
3. 输出阶段报告 / 结果摘要 / 行动清单

只要这 3 件事跑通，且能稳定演示、截图、录屏、留 Session ID，就已经具备初赛可交付基础。

### 作品名称
分享会筹备助手

### 一句话简介
一个把“准备分享会”这类模糊目标自动拆解成可执行步骤，并输出行动清单的 AI 工具。

### 我想解决的问题
很多人临近要做分享会时，会同时面对选题、内容准备、时间安排、设备检查、预演等很多事情，但不知道第一步应该做什么，也不知道怎样把这些事情排出顺序。

结果往往是：脑子里很乱，准备过程低效，临近开始时更容易慌。

我想做一个工具，在用户输入“我要准备一场分享会，但我不知道怎么安排”之后，自动帮助他完成任务拆解、步骤安排和结果汇总，让模糊目标快速变成具体行动。

### 核心方案
这个作品会围绕一个非常清晰的流程展开：
- 用户输入分享会准备目标
- 系统识别这是一个活动筹备类任务
- 自动拆成可执行步骤
- 输出阶段重点和最终行动清单

它不是单纯聊天，而是更强调执行流、结构化拆解和结果交付。

### 目标用户 / 适用场景
目标用户是需要临时准备分享、汇报、交流活动的人，比如学生、职场人、个体创作者，或任何第一次做小型分享会的人。

典型场景包括：
- 准备一次班级分享
- 准备一次工作汇报
- 准备一次小型线下交流
- 准备一次主题演讲

### 为什么想做这个
因为“准备分享会”是一个很典型、很真实、又很容易让人卡住的场景。很多人不是没有内容，而是不知道怎么把一场分享真正准备出来。

我希望这个作品能帮助用户降低准备门槛，让原本混乱的事情更快变成清晰计划。

### 预期效果
用户在输入目标后，可以快速得到：
- 一份结构化准备任务清单
- 一个更清楚的执行顺序
- 一份可直接照着做的行动清单
