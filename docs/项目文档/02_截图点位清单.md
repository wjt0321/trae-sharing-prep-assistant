# 截图点位清单｜分享会筹备助手

## 截图 1：首页首屏

### 建议画面

- 展示产品名、状态标签、主标题、副标题
- 输入框内放推荐演示输入
- 保留示例按钮和主按钮

### 截图目的

- 让评委第一眼看懂产品是干什么的
- 强化“这是一个真实产品首页”，不是临时工具页

## 截图 2：处理中状态

### 建议画面

- 点击 `开始拆解` 后的 loading 状态
- 保留“正在拆解准备步骤...”和 skeleton 卡片

### 截图目的

- 体现系统不是静态海报，而是真的在工作
- 增强录屏中的过程感

## 截图 3：阶段任务卡片区

### 建议画面

- 4 张阶段卡片完整出现
- 尽量截到标题区和两列卡片布局

### 截图目的

- 突出“系统完成了结构化任务拆解”
- 回应评委对“是不是聊天套壳”的疑问

## 截图 4：行动清单与提醒区

### 建议画面

- 左侧行动清单编号完整可见
- 右侧“准备重点 / 容易遗漏的事”同时可见

### 截图目的

- 突出结果已经收口，可直接照着做
- 体现产品不是只给想法，而是给执行结果

## 截图 5：全页合成图

### 建议画面

- 从 Hero 区到结果区完整入镜
- 优先桌面端宽屏截法

### 截图目的

- 用于比赛帖封面后的正文展示
- 展示产品完整度与页面质感

## 补充建议

- 截图时统一使用推荐输入：`我要准备一场小型分享会，但我不知道该怎么安排。`
- 如果发帖平台支持多图，推荐顺序：首屏 -> 阶段卡片 -> 行动清单 -> 全页图
- 录屏时可以先停留首屏 2 秒，再点击进入 loading 和结果页
