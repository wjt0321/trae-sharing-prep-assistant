# 分享会筹备助手 Demo PRD（给 TRAE 直接开做）

## 1. 项目目标
做一个适合 `TRAE AI 创造力大赛` 初赛提交的最小可体验 Demo：**分享会筹备助手**。

这个 Demo 的目标不是做一个大而全的 AI 助手，而是做一个**能在几分钟内看懂价值、能真实体验、能完整演示闭环**的小产品。

核心价值：
- 用户输入一个分享会准备目标
- 系统自动拆解筹备任务
- 系统输出准备顺序与阶段结果
- 用户最终得到一份可执行的行动清单

## 2. 比赛约束
这个项目是参赛作品，不是普通练手项目，必须满足以下约束：
- 必须适合初赛提交
- 必须能做成可体验 Demo
- 必须便于截图、录屏、展示
- 必须能沉淀创作过程材料
- 不要做成复杂平台
- 不要做成泛化聊天机器人
- 不追求功能堆砌，优先追求“有过程、有体验、讲得清楚”

## 3. 产品一句话定义
分享会筹备助手：一个把“准备一场分享会”这类模糊目标自动拆解成可执行步骤，并输出阶段结果与行动清单的执行型 AI 工具。

## 4. 目标用户
- 不知道怎么准备一场分享会的人
- 需要快速整理活动准备步骤的人
- 学生、职场人、个体创作者
- 第一次做小型分享、汇报、交流活动的人

## 5. 典型使用场景
本版优先只支持 1 个最容易演示的主场景：
1. 准备一场小型分享会

演示输入固定优先使用：
“我要准备一场小型分享会，但我不知道该怎么安排。”

## 6. 核心功能范围（MVP）
MVP 只做 3 个核心功能：

### 6.1 输入分享目标
用户输入一个自然语言的分享准备目标。
示例：
- 我要准备一场小型分享会，但我不知道该怎么安排
- 我要做一次 10 分钟的主题分享

### 6.2 自动筹备拆解
系统根据目标输出：
- 筹备任务列表
- 每个任务的简短说明
- 推荐准备顺序
- 可选的阶段划分（如选题 / 内容准备 / 预演 / 现场检查）

### 6.3 输出结果摘要
系统最终输出：
- 阶段性重点
- 一份行动清单
- 可直接阅读的结果摘要

## 7. 非目标（本版不要做）
本版不要做以下内容，避免失控：
- 多用户系统
- 登录注册
- 长期数据持久化系统
- 团队协作
- 日历深度集成
- 提醒推送
- 复杂权限系统
- 复杂知识库
- 语音、视频、多模态大功能
- 无边界连续 Agent 系统

## 8. 推荐页面结构
做成一个单页 Web Demo 即可，结构尽量简单清楚。

### 页面区块 1：标题区
显示：
- 产品名：分享会筹备助手
- 一句话说明
- 一个简短副标题：把分享准备变成行动

### 页面区块 2：输入区
包含：
- 一个输入框 / 文本框
- 一个示例按钮或预置示例
- 一个“开始拆解”按钮

### 页面区块 3：任务拆解结果区
展示：
- 任务 1、任务 2、任务 3...
- 每个任务的说明
- 推荐顺序
- 阶段划分（如选题阶段 / 准备阶段 / 预演阶段 / 现场检查阶段）

### 页面区块 4：结果摘要区
展示：
- 本次准备建议
- 一份行动清单
- 一段总结说明

### 页面区块 5：演示提示区
可选展示：
- 当前示例场景
- 推荐如何继续执行

## 9. 交互流程
### 主流程
1. 用户输入目标
2. 点击“开始拆解”
3. 系统显示处理中状态
4. 输出结构化任务列表
5. 输出结果摘要与行动清单

### 体验要求
- 交互响应清楚
- 输出排版清楚
- 一眼能看懂“输入了什么、拆成了什么、最后给了什么结果”
- 整个流程适合录屏和截图

## 10. 内容生成要求
AI 输出内容时，要尽量满足以下要求：
- 任务拆解要具体，不要空话
- 步骤数量控制在 5-8 步左右，避免太散
- 每步描述简洁、可执行
- 最终总结要像用户真能拿去做事的结果
- 避免输出泛化鸡汤式建议

## 11. UI / 视觉要求
- 风格简洁、清爽、现代
- 不要花里胡哨
- 优先清晰可读
- 卡片式展示任务列表
- 重点信息突出
- 适合截图展示
- PC 端优先，移动端自适应可选

## 12. 技术实现建议
可自由选择最省事、最稳的方式，但优先：
- 单页前端应用
- 可直接本地运行或快速部署
- 假数据 / 规则生成 / AI 生成均可，只要体验顺畅
- 优先保证可演示，不要求工程完美

如果需要更快落地：
- 前端可直接做静态页面 + 本地逻辑
- 如果接 AI 能力更顺，就接简单调用
- 如果接真实模型太费时间，可先做稳定可演示版本

## 13. 演示脚本要求
请在产出中同步考虑演示脚本，确保可以这样演示：

### 演示输入
“我要准备一场小型分享会，但我不知道该怎么安排。”

### 演示输出应体现
- 系统识别出这是一个活动准备类目标
- 自动拆出关键任务，如主题确定、内容准备、时间安排、场地设备、预演检查等
- 给出推荐顺序
- 最终生成一份行动清单

### 演示结论
让评委一眼感受到：
这个工具不是在聊天，而是真的把一个模糊目标转成了可以执行的步骤。
