# 容易扣分的点与避坑清单｜灵感到执行 AI 管家

## 一、最容易被误判成“普通聊天套壳”
### 风险
如果页面主体只是输入框 + 一大段回答，评委很容易觉得这只是一个聊天机器人换皮。

### 规避方法
- 一定要做结构化结果区
- 一定要有任务拆解区和执行计划区
- 页面重点放在“方案输出”，不是“聊天记录”

## 二、概念太大，场景太散
### 风险
如果一上来就说什么都能做，容易显得不聚焦、不真实。

### 规避方法
- 初赛演示只讲一个强场景
- 推荐优先讲“比赛准备”这个场景
- 其他场景作为扩展，不作为主线

## 三、没有真正的可执行感
### 风险
如果输出内容只是抽象建议，看起来不够落地。

### 规避方法
- 输出里一定要有阶段、顺序、优先级、下一步
- 最好有 3 天 / 7 天计划
- 让结果看起来像“可执行方案”而不是“灵感整理”

## 四、产品感不足
### 风险
如果页面像一个临时工具页，评委会觉得完成度不够。

### 规避方法
- 页面模块清晰
- 标题、卡片、层次、重点样式都要统一
- 页面第一屏就要能看懂产品定位

## 五、AI 价值不突出
### 风险
如果看不出 AI 真正参与了什么，项目说服力会下降。

### 规避方法
- 强调 AI 完成了目标理解、任务拆解、执行规划、动态优化
- 演示中体现“用户改条件后，方案也随之变化”

## 六、比赛表达不够强
### 风险
项目本身不错，但介绍方式太平，容易被淹没。

### 规避方法
- 用更强的一句话定位
- 开头就抛出真实痛点
- 让项目名字、口号和页面表达保持统一

## 七、真实性材料准备不足
### 风险
赛事要求保留创作过程，如果后面补不齐，会很麻烦。

### 规避方法
- 从现在开始保存关键截图
- 记录关键 Session ID
- 保留各阶段版本页面和过程说明
