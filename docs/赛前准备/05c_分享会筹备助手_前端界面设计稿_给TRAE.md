# 分享会筹备助手｜前端界面设计稿（给 TRAE）

## 1. 这份设计稿是干什么的
这不是新的产品方案，而是给 `分享会筹备助手` Demo 配套的**前端界面设计说明**。

目标不是做复杂后台，而是做一个：
- **第一眼就觉得干净、舒服、像真实产品**
- **适合比赛截图和录屏**
- **前端实现难度不要太高**
- **尽量避免 AI 味很重的“套壳网页”**

## 2. 整体视觉方向
整体风格建议采用：

**Warm Editorial + Minimal Product**

也就是：
- 有一点温暖感
- 有一点纸质感
- 但仍然是现代 Web 产品界面
- 不做炫技大渐变
- 不做很重的玻璃拟态
- 不做典型 AI Dashboard 蓝紫色

要做出的感觉接近：
- Anthropic / Claude 那种温和中性色
- 一点点编辑感（editorial）
- 一点点产品感（product）
- 干净、克制、成熟

## 3. 必须避免的视觉坑
请明确避开这些常见 AI 味前端问题：
- 不要整页紫粉蓝大渐变
- 不要满屏玻璃拟态
- 不要到处 emoji 装饰
- 不要 feature 卡片全部长一个样
- 不要堆很多花哨图标
- 不要伪造很多假数据指标卡
- 不要做成“普通后台管理系统”
- 不要做成“聊天机器人页面”

这个产品不是聊天页，也不是 dashboard，它更像一个**任务拆解工作台**。

## 4. 推荐设计关键词
给设计和实现时统一用这些关键词：
- 温暖
- 克制
- 干净
- 高级
- 轻产品感
- 易读
- 可截图
- 单页演示友好

## 5. 推荐色彩系统
建议使用暖中性色 + 琥珀强调色。

### 主背景
- 页面背景：`#F8F6F1`
- 卡片背景：`#FFFFFF`
- 次级浅底：`#F1ECE4`

### 文字
- 主文字：`#2B2926`
- 次文字：`#5F5A53`
- 弱文字：`#8C857B`

### 强调色
- 主强调色：`#C96A3D`
- Hover / 深一点强调：`#B85B31`
- 强调浅底：`rgba(201,106,61,0.10)`

### 辅助语义色
- 成功 / 完成：`#6F907C`
- 提醒 / 注意：`#B69042`
- 风险 / 警示：`#B45A42`
- 信息 / 冷静辅助：`#6C829A`

### 边框
- 主边框：`rgba(43,41,38,0.10)`
- 强边框：`rgba(43,41,38,0.16)`

## 6. 字体建议
优先使用系统里容易拿到、效果稳定的组合：

### 首选
- 中文正文：`HarmonyOS Sans SC` / `Microsoft YaHei UI` / `PingFang SC` / sans-serif
- 英文与数字：沿用同一无衬线即可
- 等宽：`JetBrains Mono` / `Consolas`

### 风格建议
- 页面标题：600-700 字重
- 正文：400-500 字重
- 说明文字：400 字重

不要用太花的 display 字体，避免实现复杂和兼容性问题。

## 7. 页面结构建议
整个页面做成一个**单页工作台**，不要多路由跳来跳去。

### 建议结构
1. 顶部 Header
2. Hero 输入区
3. 任务拆解结果区
4. 行动清单 / 总结区
5. 底部轻说明区

## 8. 页面布局细化

### 8.1 Header
顶部只保留非常少的信息：
- 左侧：产品名 `分享会筹备助手`
- 右侧：一个很轻的状态标签，例如 `Demo` / `TRAE 初赛作品`

Header 高度不要太高，风格要轻。

### 8.2 Hero 输入区（最重要）
这是页面第一视觉重点。

建议做法：
- 一个大标题
- 一句副标题
- 一个比较大的输入框
- 一个主按钮：`开始拆解`
- 一个示例输入 chip 或按钮

#### 标题文案建议
主标题：
`把分享准备变成行动清单`

副标题：
`输入一个分享目标，系统会帮你拆解出准备步骤、执行顺序和最终行动清单。`

#### 输入区视觉建议
- 输入框宽而不高，支持多行
- 按钮明显，但不要俗气
- 输入区整体放在一张大卡片里
- 卡片圆角 16px 左右
- 阴影极轻

### 8.3 任务拆解结果区
结果区不要做成普通聊天气泡。

应该做成：
- **分阶段卡片**
- 每个阶段下面列出任务
- 每条任务简短可执行

建议分 4 个阶段：
1. 目标与主题确认
2. 内容准备
3. 时间与设备安排
4. 预演与收尾检查

每个阶段卡片包含：
- 阶段标题
- 一句阶段说明
- 2-3 条具体任务

#### 卡片风格
- 白底
- 细边框
- 12-14px 圆角
- 上下节奏清晰
- 阶段标题略强调
- 任务列表不要太密

### 8.4 行动清单 / 总结区
这个区域是第二个截图重点。

建议做成：
- 左侧：最终行动清单
- 右侧：本次准备建议 / 风险提醒

或者上下布局也可以，但最好形成一种“结果已经收口”的感觉。

#### 建议内容块
- `最终行动清单`
- `准备重点`
- `容易遗漏的事`

这里的内容要像真的可以照着做，而不是泛泛总结。

### 8.5 底部轻说明区
非常轻即可：
- 一句产品说明
- 一句“本 Demo 使用 TRAE 构建”之类的小字

不要堆 footer 导航。

## 9. 推荐视觉层次
页面要形成 3 个明显层次：

### 第一层：主输入
用户一上来先看这里

### 第二层：任务拆解结果
体现产品的主要价值

### 第三层：最终行动清单
体现结果收口和实用性

不要让页面所有区域视觉权重一样。

## 10. 推荐组件风格

### 按钮
- 主按钮使用暖琥珀色
- 圆角 10px
- 字重 500-600
- Hover 只做轻微变深
- 不要发光特效

### 输入框
- 白底
- 细边框
- Focus 时边框变暖色
- 适当留白
- placeholder 文字偏浅

### 卡片
- 白底
- 轻边框
- 很浅的阴影
- 圆角 14-16px

### 标签 / Chip
可用于：
- 示例输入
- Demo 标记
- 阶段标签

风格要轻，别太像后台筛选条件。

## 11. 推荐交互状态

### 初始态
- 页面很干净
- 输入区在视觉中心
- 结果区可以是空态提示

### 处理中状态
- 不要只转圈
- 最好有轻微 skeleton 或“正在拆解准备步骤...”文字
- 让用户感觉系统真的在工作

### 输出态
- 平滑显示各阶段卡片
- 行动清单区同时出现
- 可以有轻微淡入动画

### 动画原则
- 150-250ms 即可
- 只做 opacity / translateY 级别微动效
- 不做大幅滑动和炫酷转场

## 12. 响应式建议
比赛主要看桌面端，但移动端不要完全崩。

### 桌面端优先宽度
- 页面主体最大宽度：`1120px` 左右
- 输入区稍宽
- 结果区可做 2 列或 12 栏栅格

### 移动端
- 所有内容单列
- 输入框和按钮纵向堆叠也可以
- 卡片保持大留白

## 13. 推荐技术实现路线
为了让 TRAE 更容易做出来，建议：

### 最省事方案
- React + Tailwind CSS

### 组件策略
- 自定义少量卡片、按钮、输入框即可
- 不一定非要上复杂 UI 库

### 如果要加组件库
优先考虑：
- `shadcn/ui`（适合快速做高级感基础组件）
- 图标用 `lucide-react`

### 但注意
不要让组件库默认风格原封不动出现，至少要覆盖色彩、圆角、边框和间距，不然会很模板味。

## 14. 推荐参考风格来源
结合当前本地可借鉴方向，建议参考这两类：

### 方向 A：Claude / Anthropic 风格
借鉴点：
- 暖中性色
- 安静克制
- 留白舒服
- 不像普通 SaaS 后台

### 方向 B：Open Design / Editorial Product 风格
借鉴点：
- 有设计感但不花哨
- 结构化很强
- 容易做成比赛截图

## 15. 给 TRAE 的页面实现要求
请按以下要求实现前端：
- 先做单页
- 先做桌面端
- 先做初始态、处理中、输出态 3 个状态
- 优先把输入区和结果区做漂亮
- 不要花太多时间在复杂导航和多页面结构上
- 不要做普通聊天框 UI
- 不要做后台管理页面 UI

## 16. 给 TRAE 的最简设计指令
如果需要一句话压缩给 TRAE：

**请把这个 Demo 做成一个温暖、克制、像真实产品的单页工作台，视觉风格参考 Claude / Anthropic 的暖中性色界面，但保留更强的任务拆解结果展示。不要做成聊天机器人页面，也不要做成普通后台。重点把输入区、阶段任务卡片、行动清单做漂亮，适合截图和录屏。**

## 17. 最终验收标准
前端只要满足这些，就算合格：
- 第一眼好看，不土
- 不像 AI 套壳网页
- 不像后台管理系统
- 输入区有产品感
- 结果区结构清楚
- 行动清单区适合截图
- 整页适合比赛展示
