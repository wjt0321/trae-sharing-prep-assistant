# 分享会筹备助手｜给 TRAE 的执行提示词

请你直接帮我实现一个适合 `TRAE AI 创造力大赛` 初赛提交的最小可体验 Demo，产品名称叫：**分享会筹备助手**。

## 你要做的产品是什么
这是一个单场景的小工具，不是万能 AI 助手。

它专门解决这个问题：
**“我要准备一场小型分享会，但我不知道怎么开始。”**

用户输入一个分享目标后，系统需要：
- 自动拆解出筹备任务
- 给出准备顺序
- 输出阶段结果
- 最后生成一份可直接照着做的行动清单

## 这个 Demo 的核心目标
不要做大，不要做复杂。
你要优先做出一个：
- 能打开
- 能输入目标
- 能输出结构化结果
- 能清楚演示价值
- 能截图、录屏、发比赛帖的小产品

## MVP 只做 3 件事
1. 输入分享准备目标
2. 自动拆解筹备任务
3. 输出行动清单和结果摘要

## 推荐演示输入
请优先围绕这个输入做演示体验：
`我要准备一场小型分享会，但我不知道该怎么安排。`

## 推荐输出方向
输出内容请尽量包含：
- 选题或目标确认
- 内容准备
- 时间安排
- 设备 / 现场检查
- 预演与收尾

并整理成：
- 分阶段任务列表
- 推荐顺序
- 一份最终行动清单

## 页面怎么做
请做成一个单页 Web Demo，结构尽量简单清楚。

建议页面包含：
- 顶部标题区（产品名 + 一句话说明）
- 输入区（输入框 + 示例按钮 + 开始按钮）
- 任务结果区（任务卡片 / 阶段拆解）
- 结果摘要区（行动清单 + 总结）

## 视觉要求
- 简洁、清爽、现代
- 不要花里胡哨
- 适合截图展示
- 一眼能看懂“输入了什么、输出了什么”

## 交互要求
- 用户输入目标后，点击按钮开始拆解
- 页面展示处理中状态
- 然后输出结构化任务结果
- 输出排版要清楚，适合录屏和比赛展示

## 内容要求
请注意：
- 输出不要空泛，不要鸡汤
- 每个步骤要短、清楚、可执行
- 总步骤数控制在 5-8 步左右
- 最终结果要像一个人真的可以照着做

## 明确不要做的事
不要做这些：
- 不要做万能聊天助手
- 不要做复杂平台
- 不要做登录注册
- 不要做多人协作
- 不要做复杂数据库系统
- 不要做无边界 Agent
- 不要为了炫技堆太多功能

## 比赛导向要求
这个项目是为了比赛，不是普通练手项目，所以请优先满足：
- 演示闭环清楚
- 价值容易理解
- 页面适合截图
- 结果适合录屏
- 能作为一个真实小工具展示

## 交付时请一并考虑
请尽量同步产出或方便我整理出这些内容：
- 首页效果图
- 一组结果页截图点位
- 简短项目说明
- 可用于比赛帖的作品简介

## 最终标准
只要最终结果满足下面这些，就算成功：
- Demo 能运行
- 目标输入后能生成任务拆解
- 能输出行动清单
- 页面看起来像真实产品
- 适合比赛演示和发帖

请直接开始实现，不要先扩展范围，不要先做复杂架构，先把最小可运行 Demo 做出来。
