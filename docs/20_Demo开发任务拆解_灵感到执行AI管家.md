# Demo 开发任务拆解｜灵感到执行 AI 管家

## 开发目标
在初赛阶段完成一个可交互、可演示、可体现 AI 核心价值的网页 Demo。

## 一、产品层任务
### 1. 明确 MVP 边界
- 只做一条主路径
- 聚焦“输入目标 → 拆解任务 → 生成计划 → 输出结果 → 二次优化”
- 不做复杂账号系统、数据库、多人协作等重功能

### 2. 明确主场景
- 主场景固定为：AI 创造力大赛准备
- 其它场景仅作为扩展示例，不作为核心演示路径

### 3. 明确展示重点
- 任务拆解要清楚
- 执行计划要像真的能照着做
- 结果输出要结构化
- 二次优化要能体现 AI 的动态调整能力

## 二、页面层任务
### 1. 首页框架
- 放项目名、口号、说明文案
- 放一个示例目标按钮
- 放主输入区

### 2. 输入区
- 输入目标文本框
- 补充条件输入框
- 生成方案按钮

### 3. 任务拆解区
- 做成卡片列表
- 每张卡片展示：任务名、阶段、优先级、简要说明

### 4. 执行计划区
- 用 3 天 / 7 天视图展示推进路径
- 每天列出关键任务与目标

### 5. 结果输出区
- 今日重点
- 当前阶段目标
- 风险提醒
- 下一步建议

### 6. 二次优化区
- 增加一个“补充要求”输入框
- 点击后刷新任务与计划结果

## 三、内容层任务
### 1. 准备默认示例输入
- 比赛准备
- 工作任务
- 学习目标

### 2. 准备任务拆解示例内容
- 至少 1 套高质量大赛场景数据
- 保证演示时输出稳定、好看、合理

### 3. 准备结果区文案
- 今日重点
- 阶段目标
- 风险提示
- 下一步建议

## 四、交互层任务
### 1. 生成流程
- 用户输入目标
- 展示加载态
- 输出完整结果

### 2. 二次优化流程
- 用户补充一句条件
- 页面局部刷新结果
- 明确告诉用户哪些内容被重新优化

## 五、演示层任务
### 1. 准备默认演示案例
“我要在 1 周内完成 AI 创造力大赛报名和 Demo，时间有限，希望更适合比赛评审。”

### 2. 准备追加条件
“希望它更像正式产品，展示更有产品感。”

### 3. 保证演示时长
- 整体控制在 3 分钟左右
- 页面切换逻辑尽量少
- 重点展示结果，不展示杂乱过程

## 六、完成标准
- 页面可打开
- 主流程可完整演示
- 结果内容结构清晰
- 二次优化能成功展示
- 页面看起来像产品，而不是临时拼出来的 demo
