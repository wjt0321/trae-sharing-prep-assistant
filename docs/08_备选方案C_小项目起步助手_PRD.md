# 备选方案 C｜小项目起步助手 Demo PRD

## 1. 这是干什么的
这是一个把“我要做一个小项目，但不知道怎么开始”自动拆成起步步骤的小工具。

它不是项目管理系统，也不是 PM 文档生成器，而是帮助用户完成“从想法到启动”的第一段路。

## 2. 核心场景
用户输入：
- 我想做一个校园活动报名小程序
- 我想做一个个人副业小项目
- 我想尝试做一个小型工具产品，但不知道先做什么

系统输出：
- 项目起步步骤
- 最小闭环建议
- 第一周行动清单
- 风险提醒

## 3. 为什么它适合比赛
- 更有生产力工具气质
- 很适合和 TRAE 的创作语境结合
- 既有现实价值，也容易做成 Demo

## 4. MVP 只做什么
- 输入项目目标
- 输出最小闭环建议
- 输出起步步骤
- 输出第一周行动清单

## 5. 不做什么
- 不做完整项目管理平台
- 不做多人协作
- 不做甘特图
- 不做复杂知识库
- 不做代码仓库集成

## 6. 推荐页面结构
- 顶部标题：小项目起步助手
- 输入区：输入项目想法
- 起步区：先做什么
- 闭环区：最小版本建议
- 行动区：第一周清单

## 7. 推荐演示输入
“我想做一个校园活动报名工具，但我不知道先做什么。”

## 8. 推荐演示输出
- 先明确目标用户
- 先定义核心流程
- 先做报名 + 查看信息两个核心功能
- 先验证最小使用场景
- 输出第一周行动计划

## 9. 交付要求
- 单页可体验 Demo
- 重点体现“先做什么”
- 输出不要空，要能真落地
- 保留过程截图和 Session ID

## 10. 成功标准
- 用户能输入项目想法
- 系统能输出可执行起步步骤
- 看起来像真实项目启动助手
- 适合初赛演示
